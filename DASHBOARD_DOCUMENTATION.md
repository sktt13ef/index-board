# 黄嘻得 A股看板 - 数据更新逻辑说明

## 一句话总结

**所有数据都不是写死的，全部通过 API 动态获取 + 本地缓存。**

## 数据流程

```
用户打开网页
    │
    ▼
Flask 后端接收请求
    │
    ├── 实时行情 → 调用东方财富API → 返回最新数据
    │                        └── 保存JSON缓存（下次直接用）
    │
    ├── 历史K线 → 检查本地CSV是否存在
    │            ├── 存在且未过期 → 直接返回CSV
    │            └── 过期或不存在 → 从远程API获取 → 保存CSV → 返回
    │
    └── 估值数据 → 调用新浪财经获取美股期货 → 公式计算 → 返回
```

## 手动更新数据

```bash
# 查看数据信息
python update_history_data.py --info

# 更新过期数据
python update_history_data.py

# 强制更新所有数据
python update_history_data.py --force
```

## 数据文件位置

```
stock_dashboard/data/
├── data.json                ← 实时行情缓存
├── index_data.json          ← 指数行情缓存
├── fund_net_values.json     ← 基金净值缓存
├── valuation_data_realtime.json  ← 实时估值数据
└── history/                 ← 历史K线目录
    ├── CSI300.csv           ← 沪深300历史
    ├── CSI500.csv           ← 中证A500历史
    ├── HSTECH.csv           ← 恒生科技历史
    ├── ...
    └── meta.json            ← 各数据更新时间记录
```

## 关键参数

- **数据有效期**：24小时
- **有效期检查逻辑**：`当前时间 - last_update < 24小时`
- **存储格式**：CSV（历史K线）+ JSON（实时行情）
