# 市场看板数据核查报告

核查时间：2026-06-30 11:49:35 +08:00  
核查范围：当前卡片行情、10 年历史曲线、指数收益/全收益曲线、估值来源、前端展示口径。

## 结论

当前卡片行情的 20 个标的都能返回数据，且 `当前价 - 昨收 = 涨跌额` 的内部一致性全部通过。

但“所有数据准确无误”目前不能这样下结论。主要问题在历史图和全收益口径：

1. 运行中的页面 API 只有 3 组红利低波指数能返回真实全收益曲线，其它标的的 `series=total_return` 当前返回空数组。
2. 源码里的最新历史源配置已经能直接拉到大多数标的的价格曲线和替代全收益曲线，但运行中的 Flask 进程没有完全加载这些新配置。
3. 本地 CSV 缓存存在旧口径数据，例如 HSI / HSTECH / DAX 的缓存记录数和当前源码标注的来源不完全一致，需要做缓存版本管理或重建缓存。
4. 部分“全收益”不是官方 total return index，而是 ETF / 同源价格代理，必须在页面上明确标注，不能伪装成官方全收益。

## 估值数据核查

估值区块不是所有标的都有真实 PE。当前可信口径如下：

| 类型 | 标的 | 来源 | 说明 |
|---|---|---|---|
| 中证 PE | CSI300、CSI500、红利低波系列、中证白酒、科创系列 | 中证指数官网 `indexCsiDsPe` | 官方日频 PE，可用于十年分位 |
| 标普 500 PE | SPX | multpl.com 月频 PE-TTM | 非交易所直连，属于公开估值数据源 |
| 收益率分位 | CN10Y、US10Y | 官方收益率曲线历史 | 这里是收益率历史分位，不是 PE |
| 价格分位 | HSI、HSTECH、DAX、NDX、GOLD、OIL_WTI、XOP、WANJIA_GOLD | 对应历史价格源 | 不是估值 PE，只能表达价格所处历史位置 |

## 需要修复的地方

1. 重启 Flask 服务，确保运行态加载当前 `history_data_manager.py` 里的最新 series 配置。
2. 清理或重建 `data/history/*.csv`，尤其是 HSI、HSTECH、DAX 这类源配置变更过的缓存。
3. 给历史缓存加 `source_id + source_symbol + schema_version`，当源码切换来源时自动失效，避免旧 CSV 被当成新来源数据。
4. `/api/history` 返回值应附带 `history_source_label`、`history_source_url`、`series_label`，前端不能只显示卡片来源。
5. “全收益”按钮需要区分三种状态：官方全收益、ETF/基金代理、无全收益意义。商品和 DAX/XOP 这类同源曲线不应简单叫“全收益”。
6. 如果坚持所有标的都有“两条线”，建议按钮文案改为 `指数收益 / 代理收益 / 两条线`，并在 tooltip 或图例里标注代理来源。

## 最终判断

卡片行情：当前可用，内部一致性通过。  
走势图：当前运行态不完整，尤其全收益大面积为空。  
源码数据源：大多数已经能找到免费可拉取来源，但有若干代理口径。  
要达到“真实、及时、可追溯”，下一步必须做缓存版本管理和历史图来源标注，否则用户看图时仍可能误以为全收益都是官方口径。
